## 1. Hook

In 1991, a Patriot missile failed to intercept an Iraqi Scud because of a floating-point timing error. 28 soldiers died. The error was 0.000000095 seconds per hour — but over 100 hours, it added up to a 0.34-second targeting error.

## 2. Why 0.1 + 0.2 ≠ 0.3

The number 0.1 in binary is infinitely repeating:
```
0.1 = 0.00011001100110011... (repeating)
```

Just like 1/3 (0.3333...) can't be represented exactly in decimal, 0.1 can't be represented exactly in binary.

### The Result

```javascript
0.1 + 0.2 = 0.30000000000000004
```

## 3. Handling Floating Point

```javascript
// Instead of:
if (a + b === 0.3)

// Use epsilon comparison:
function nearlyEqual(a, b, epsilon = 0.0001) {
  return Math.abs(a - b) < epsilon;
}
```

## 4. Summary

- Some decimal numbers have **infinite binary representations**
- C**ompare with epsilon**, not equality
- For financial calculations, use **fixed-point** or integers
