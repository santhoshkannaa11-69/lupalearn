## 1. Hook

How do you represent -5 in binary? You can't just put a minus sign in front — computers only have 0s and 1s. The most common solution is **two's complement**, and it's elegant.

## 2. Three Approaches

| Method | Description | Problem |
|--------|-------------|---------|
| **Sign-magnitude** | First bit = sign, rest = value | Two zeros (+0 and -0) |
| **One's complement** | Invert all bits | Still two zeros |
| **Two's complement** | Invert + add 1 | One zero, works with addition |

## 3. Two's Complement

### How It Works

```
Positive 5:   00000101
Invert bits:  11111010
Add 1:        11111011  (= -5 in two's complement)
```

### Range

| Bits | Signed Range |
|------|-------------|
| 8 | -128 to 127 |
| 16 | -32,768 to 32,767 |
| 32 | ~-2.1 billion to ~2.1 billion |

function toTwosComplement(value, bits) {

## 5. Summary

- **Two's complement** is the standard way to represent signed integers
- The MSB indicates the sign (1 = negative)
- Range is asymmetrical: **-2ⁿ⁻¹ to 2ⁿ⁻¹ - 1**
