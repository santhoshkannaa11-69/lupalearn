## 1. Hook

When you save a file, it probably won't be stored in one contiguous chunk. Your 10 KB file might be split across 10 different locations on the disk. The file system keeps a map of where all the pieces are.

## 2. Blocks and Clusters

| Term | Definition | Typical Size |
|------|------------|-------------|
| **Block** | Smallest unit the storage device uses | 512 bytes - 4 KB |
| **Cluster** | Smallest unit the file system allocates | 4 KB - 64 KB |
| **Fragment** | A file split across non-contiguous clusters | Depends on fragmentation |

## 3. Fragmentation

```
Healthy: [File A: ████████] [File B: ████████] [Free: ████████]
                        ↓
Fragmented after edits/deletes:
         [File A: ██░░██░░██] [Free: ░░░░████░░░░]
```

HDDs slow down significantly when fragmented. SSDs don't care (same access time anywhere).

## 4. SSD vs HDD for File Storage

| Aspect | HDD | SSD |
|--------|-----|-----|
| **Read speed** | ~150 MB/s | ~5,000 MB/s |
| **Fragmentation** | Major performance issue | Not an issue |
| **Data persistence** | Magnetic charge | Electrical charge (fades over years) |
| **Failure mode** | Gradual (bad sectors) | Sudden (controller failure) |

## 5. Summary

- Files are stored in **blocks/clusters**
- **Fragmentation** slows HDDs but doesn't affect SSDs
- The **file system maintains a map** of where each file's data lives
