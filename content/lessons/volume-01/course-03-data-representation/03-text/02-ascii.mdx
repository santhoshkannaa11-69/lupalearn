## 1. Hook

The letter 'A' is stored as 65. 'a' is 97. The difference between uppercase and lowercase is exactly 32. This wasn't random — it was designed so that flipping one bit changes case.

## 2. ASCII Table (7-bit, 128 characters)

| Code Range | Category | Examples |
|-----------|----------|----------|
| 0-31 | Control characters | Null, Bell, Tab, Enter |
| 32 | Space | |
| 48-57 | Digits | 0-9 |
| 65-90 | Uppercase | A-Z |
| 97-122 | Lowercase | a-z |

### Key Values to Remember

| Character | Decimal | Binary |
|-----------|---------|--------|
| A | 65 | 01000001 |
| a | 97 | 01100001 |
| 0 | 48 | 00110000 |

## 3. ASCII's Limitation

128 characters was enough for English, but not for:
- Accented characters (é, ñ, ü)
- Non-Latin scripts (Cyrillic, Arabic, Chinese)
- Emojis

## 5. Summary

- **ASCII** uses 7 bits for 128 characters
- Designed for English — uppercase, lowercase, digits, punctuation
- Insufficient for global computing
