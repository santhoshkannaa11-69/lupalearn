## 1. Hook

Over 1 million characters can be represented in Unicode — every writing system in the world, past and present, plus emojis, mathematical symbols, and more. And UTF-8 packs all of that while remaining backward-compatible with ASCII.

## 2. Unicode vs UTF

| Layer | What It Is | Example |
|-------|-----------|---------|
| **Unicode** | A mapping of characters to code points | 'A' = U+0041 |
| **UTF-8** | How code points are stored (variable) | 1-4 bytes |
| **UTF-16** | How code points are stored (2-4 bytes) | 2-4 bytes |
| **UTF-32** | How code points are stored (fixed) | 4 bytes |

## 3. UTF-8 Encoding

| Code Point Range | UTF-8 Bytes | Example |
|-----------------|-------------|---------|
| U+0000 to U+007F | 1 byte (ASCII) | A → 01000001 |
| U+0080 to U+07FF | 2 bytes | é → 11000011 10101001 |
| U+0800 to U+FFFF | 3 bytes | 中 → 11100100 10111000 10101101 |
| U+10000+ | 4 bytes | 🎉 → 11110000 10011111 10001110 10001001 |

## 5. Summary

- **Unicode** maps characters to code points (U+XXXX)
- **UTF-8** is the dominant encoding — ASCII-compatible, variable-length
- UTF-8 powers over 98% of the web
