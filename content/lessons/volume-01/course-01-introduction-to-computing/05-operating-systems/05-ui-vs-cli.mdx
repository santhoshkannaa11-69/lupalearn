## UI vs Command Line

You can interact with your computer in two ways.

### Graphical User Interface (GUI)

Windows, icons, buttons, and menus. Intuitive and visual.

| Pro | Con |
|-----|-----|
| Easy to learn | Slower for power users |
| Visual feedback | Hard to automate |
| Discoverable | Uses more resources |

### Command Line Interface (CLI)

Typing commands in a terminal. Powerful and efficient.

| Pro | Con |
|-----|-----|
| Very fast | Steeper learning curve |
| Scriptable/automated | No visual feedback |
| Minimal resources | Requires memorizing commands |
| Accessible remotely | |

### When to Use Each

| Task | Better With |
|------|-------------|
| Browsing the web | GUI |
| Editing a photo | GUI |
| Renaming 1000 files | CLI (`rename 's/.jpg/.png/' *.jpg`) |
| Installing software | CLI (`npm install react`) |
| Configuring a server | CLI |
| Writing code | Both (editor GUI + terminal CLI) |

### In the Playground

Open the **Playground** and try the terminal. Type `ls`, `pwd`, `cd` — you're using a CLI right now. As you progress through LupaLearn, you'll become comfortable with both interfaces.

The most effective developers use **both** — choosing the right tool for each task.
