## 1. Hook

You've been using a GUI your whole life — windows, icons, buttons. But behind every GUI is a command line waiting to be used. And for many tasks, the command line is faster than any menu.

## 2. GUI vs CLI

| Task | GUI | CLI |
|------|-----|-----|
| **Rename 100 files** | Click each one → F2 | `rename 's/.jpg/.png/'` — 1 second |
| **Find a file** | Search bar | `find / -name "*.txt"` |
| **Copy to server** | FTP client | `scp file.zip server.com:/` |

## 3. Why the CLI Matters for Developers

- **Scriptable** — automate repetitive tasks
- **Available everywhere** — every server has a terminal
- **Composable** — chain commands together (`ls | grep "test" | wc -l`)
- **Lightweight** — runs on any machine, any connection speed

## 4. Summary

- **GUI** is intuitive and discoverable
- **CLI** is faster and more powerful for complex tasks
- Developers use **both** — choosing the right tool for each task
