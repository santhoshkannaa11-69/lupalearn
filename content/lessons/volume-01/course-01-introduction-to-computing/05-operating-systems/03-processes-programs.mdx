## 1. Hook

When you open Chrome, you create one process. Open 10 tabs — now you've created 10 processes. Chrome itself is the program, but each tab is a separate *running instance*.

## 2. Program vs Process

| Aspect | Program | Process |
|--------|---------|---------|
| **What it is** | File on disk | Running in memory |
| **State** | Static | Active |
| **Multiple?** | One file | Many copies can run |

## 3. Context Switching

The OS creates the illusion of multitasking by rapidly switching between processes:

```
Process A runs (10ms)
    → Save A's state
    → Load B's state
    → Process B runs (10ms)
    → Save B's state
    → Load C's state
    → ...
```

This happens thousands of times per second.

## 4. Summary

- A **program** is a static file on disk
- A **process** is a program that's currently **running** in memory
- The OS switches between processes so fast it feels like everything runs at once
