## 1. Hook

When you type `google.com`, your computer sends a message to a server — but it doesn't know where the server is. It asks a "phone book" (DNS), gets an address (IP), then sends your request in small pieces (packets) that each take their own route.

## 2. Key Concepts

| Concept | Analogy |
|---------|---------|
| **IP Address** (192.168.1.1) | Street address |
| **DNS** (google.com → IP) | Phone book |
| **Packet** (small data chunk) | Postcard |
| **Router** | Post office sorting facility |
| **Protocol** (rules) | Language |

## 3. How It Works

```
Your computer → Router → ISP → Internet backbone → Server
    ↓                                                    ↓
Sends packets → Routers forward → Routers forward → Reassembled
```

## 4. Summary

- **IP addresses** uniquely identify devices on the Internet
- **DNS** translates domain names to IP addresses
- Data is broken into **packets**, routed independently, and reassembled
- This all happens in **milliseconds**
