## 1. Hook

A program is a set of instructions that tells a computer what to do. But behind every program is a person who had to figure out *what* to tell it. That's programming — not the typing, but the thinking.

## 2. The Programmer's Workflow

```
1. Understand the problem
2. Break it down into steps
3. Design a solution
4. Write the code (implement)
5. Test that it works
6. Repeat (iterate)
```

## 3. Programming as Problem-Solving

Programming is not about memorizing syntax. It's about:

- **Breaking** complex problems into small steps
- **Recognizing** patterns from problems you've solved before
- **Testing** your assumptions early and often
- **Learning** from mistakes (most code fails on the first try)

## 4. Summary

- Programming is **problem-solving**, not typing
- The workflow: **understand → break down → design → implement → test**
- Most of programming is thinking, not coding
