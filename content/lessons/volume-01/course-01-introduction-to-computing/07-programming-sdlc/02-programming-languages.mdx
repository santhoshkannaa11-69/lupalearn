## Programming Languages Explained

A **programming language** is a formal language for writing instructions that a computer can execute.

### Levels of Languages

```
High-level (human-readable)    Python, JavaScript, Java
         ↓
Mid-level                      C, C++, Rust
         ↓
Low-level (machine-specific)   Assembly
         ↓
Machine code (binary)          10110010 01101001
```

| Level | Readability | Control | Speed |
|-------|-------------|---------|-------|
| **High-level** | Easy to read | Less control | Fast enough |
| **Mid-level** | Moderate | More control | Fast |
| **Low-level** | Hard to read | Full control | Fastest |

### High-Level Languages Compared

| Language | Best For | Key Feature |
|----------|----------|-------------|
| **Python** | Data science, AI, automation | Readability, extensive libraries |
| **JavaScript** | Web development | Runs in browsers, Node.js |
| **TypeScript** | Large web apps | Type safety, tooling |
| **Java** | Enterprise, Android | Platform independence |
| **Rust** | Systems programming | Memory safety, performance |
| **Go** | Cloud services, APIs | Concurrency, simplicity |
| **C++** | Games, engines | Performance, control |

### Compilation vs Interpretation

| Approach | How It Works | Languages |
|----------|-------------|-----------|
| **Compiled** | Source → Machine code (ahead of time) | C, C++, Rust, Go |
| **Interpreted** | Source → Execution (line by line) | Python, JavaScript |
| **JIT Compiled** | Source → Intermediate → Machine (at runtime) | Java, C# |
