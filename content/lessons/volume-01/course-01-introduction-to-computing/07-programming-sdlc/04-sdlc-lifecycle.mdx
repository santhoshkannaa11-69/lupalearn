## 1. Hook

Every software project — from a simple app to a space shuttle — follows the same basic stages. The difference is which stages you repeat and how often.

## 2. The 6 Phases

```
Requirements → Design → Implementation → Testing → Deployment → Maintenance
```

| Phase | What Happens |
|-------|-------------|
| **Requirements** | Define what the user needs |
| **Design** | Plan the architecture and UI |
| **Implementation** | Write the code |
| **Testing** | Verify it works correctly |
| **Deployment** | Release to production |
| **Maintenance** | Fix bugs, add features |

## 3. Waterfall vs Agile

| Approach | Style | When to Use |
|----------|-------|-------------|
| **Waterfall** | Sequential, one pass | Simple, well-understood projects |
| **Agile** | Iterative, continuous | Complex, evolving projects |

LupaLearn itself uses Agile — each sprint delivers working features.

## 4. Summary

- The SDLC provides a **structured framework** for building software
- **Waterfall** is linear; **Agile** is iterative
- Most modern software teams use **Agile** with 1-2 week sprints
