## The Software Development Lifecycle (SDLC)

The **SDLC** is a structured process for planning, creating, testing, and deploying software.

### The 6 Phases

```
1. Requirements ─→ What does the user need?
2. Design       ─→ How will we build it?
3. Implementation → Write the code
4. Testing      ─→ Does it work correctly?
5. Deployment   ─→ Release to users
6. Maintenance  ─→ Fix bugs, add features
```

### Phase 1: Requirements

- Gather user needs
- Define features and functionality
- Create user stories: "As a user, I want to..."
- Example: "As a student, I want to track my completed lessons"

### Phase 2: Design

- System architecture
- Database design
- User interface mockups
- Technology choices

### Phase 3: Implementation

- Write actual code
- Follow coding standards
- Use version control (Git)
- Code reviews

### Phase 4: Testing

- **Unit tests**: Test individual functions
- **Integration tests**: Test components together
- **User testing**: Real people try the software
- **Bug fixes**: Fix what's broken

### Phase 5: Deployment

- Release to production servers
- Monitor for issues
- Rollback if problems arise

### Phase 6: Maintenance

- Fix bugs discovered by users
- Add new features
- Update dependencies
- Improve performance

### Development Methodologies

| Approach | Style | When to Use |
|----------|-------|-------------|
| **Waterfall** | Sequential phases | Simple, well-understood projects |
| **Agile** | Iterative cycles | Complex, evolving projects |
| **Scrum** | 2-week sprints | Most modern software teams |
