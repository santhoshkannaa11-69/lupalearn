## How Software Is Built

Modern software is built by teams using established processes and tools.

### The Team

| Role | Responsibility |
|------|---------------|
| **Product Manager** | Defines what to build |
| **Designer** | Designs how it looks and works |
| **Developer** | Writes the code |
| **QA Tester** | Finds bugs |
| **DevOps Engineer** | Deploys and maintains |
| **Tech Lead** | Guides technical decisions |

### Essential Tools

| Tool | Purpose | Examples |
|------|---------|----------|
| **Version Control** | Track code changes | Git, GitHub |
| **IDE/Editor** | Write code | VS Code, IntelliJ |
| **Package Manager** | Manage dependencies | npm, pip, cargo |
| **CI/CD** | Automate testing/deployment | GitHub Actions |
| **Project Management** | Track tasks | Jira, Linear |
| **Communication** | Team coordination | Slack, Discord |

### The Development Cycle

```
Plan → Design → Code → Test → Deploy → Monitor → (repeat)
```

This cycle happens rapidly — sometimes dozens of times per day in modern teams.

### You Don't Need It All Yet

As a beginner, focus on:
1. A code editor (the LupaLearn Playground has one built-in)
2. Version control (Git is covered in Course 9)
3. A curious mindset

The rest comes with experience.
