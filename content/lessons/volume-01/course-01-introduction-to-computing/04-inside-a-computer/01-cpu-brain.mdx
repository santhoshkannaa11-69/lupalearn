## 1. Hook

A modern CPU executes billions of instructions per second. That's like reading every word on 10,000 books in a single second. How does it keep track of what to do next?

## 2. The Fetch-Decode-Execute Cycle

```
┌─────────┐     ┌─────────┐     ┌─────────┐
│ FETCH   │────→│ DECODE  │────→│ EXECUTE │
│ Get next │     │ Translate│     │ Perform  │
│instruction│     │to commands│     │operation │
└─────────┘     └─────────┘     └─────────┘
       ↑                           │
       └───────────────────────────┘
```

## 3. CPU Components

| Component | Purpose |
|-----------|---------|
| **Control Unit** | Fetches and decodes instructions |
| **ALU** | Performs math and logic |
| **Registers** | Ultra-fast internal memory |
| **Cache** | Small, fast buffer (L1, L2, L3) |

## 4. Cores vs Speed

| Spec | What It Means |
|------|---------------|
| **Clock speed (GHz)** | Cycles per second (higher = faster) |
| **Cores** | Independent processing units |
| **Threads** | Virtual cores (hyper-threading) |

## 5. Summary

- CPU follows **fetch → decode → execute** billions of times per second
- More **cores** = handles more tasks simultaneously
- Higher **clock speed** = processes each task faster

## 6. What's Next

The CPU needs fast access to data. Next: **RAM** — the computer's short-term memory.
