## 1. Hook

Hardware is the body. Software is the mind. A computer without software is a collection of expensive metal and silicon — it can't do anything until told what to do.

## 2. The Software Stack

```
┌──────────────────────────────┐
│    Applications              │  ← What users interact with
│  (Chrome, VSCode, Spotify)  │
├──────────────────────────────┤
│    Operating System          │  ← Manages hardware resources
│  (Windows, macOS, Linux)    │
├──────────────────────────────┤
│    Drivers                   │  ← Translates between OS and hardware
├──────────────────────────────┤
│    Firmware / BIOS           │  ← Built into hardware, boots the system
└──────────────────────────────┘
```

## 3. Types of Software

| Type | Examples | Purpose |
|------|----------|---------|
| **Operating System** | Windows, macOS, Linux | Manages resources |
| **Applications** | Chrome, VS Code, Spotify | User tasks |
| **Drivers** | GPU driver, printer driver | Hardware communication |
| **Firmware** | BIOS, router firmware | Device-level control |
| **Utilities** | Antivirus, disk cleanup | System maintenance |

## 4. How They Interact

```
User opens Chrome
  → OS allocates RAM for Chrome
  → Chrome requests network access
  → Driver sends signal to Wi-Fi card
  → Hardware transmits data
  → Result flows back up the stack
```

## 5. Summary

- **Software** is the instructions that tell hardware what to do
- The **OS** manages hardware and runs applications
- **Drivers** bridge the gap between OS and hardware
- **Firmware** provides low-level device control
