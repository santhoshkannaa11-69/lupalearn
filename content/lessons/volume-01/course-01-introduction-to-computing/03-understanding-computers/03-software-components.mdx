## Software Components

**Software** is the set of instructions that tell hardware what to do. Without software, hardware is just expensive metal and silicon.

### The Software Stack

```
User
  ↓
Applications (browser, games, editors)
  ↓
Operating System (Windows, macOS, Linux)
  ↓
Drivers (communicate with hardware)
  ↓
Hardware (CPU, memory, devices)
```

### Types of Software

| Type | What It Does | Examples |
|------|-------------|----------|
| **Operating System** | Manages hardware and resources | Windows, macOS, Linux, Android, iOS |
| **Applications** | Performs specific tasks | Chrome, Word, VS Code, Spotify |
| **Utilities** | System maintenance | Antivirus, disk cleanup, backup |
| **Drivers** | Communicate with hardware | GPU driver, printer driver |
| **Firmware** | Built into hardware | BIOS, router firmware |

### How Software Interacts with Hardware

1. An **application** requests data
2. The **OS** manages the request
3. The **driver** translates the request to hardware commands
4. The **hardware** performs the operation
5. The result flows back up the stack

This layering allows you to run complex software without understanding the underlying hardware details.
