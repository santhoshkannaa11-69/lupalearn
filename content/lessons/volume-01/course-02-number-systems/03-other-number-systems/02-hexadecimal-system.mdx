## 1. Hook

When you see `#FF5733` for a color, or `0x7FFF` for a memory address, you're looking at hexadecimal. One hex digit represents exactly 4 binary digits — making it the most compact way to display binary data.

## 2. Core Concept

### Hex Digits

| Decimal | Binary | Hex |
|---------|--------|-----|
| 0 | 0000 | 0 |
| 9 | 1001 | 9 |
| 10 | 1010 | A |
| 11 | 1011 | B |
| 12 | 1100 | C |
| 13 | 1101 | D |
| 14 | 1110 | E |
| 15 | 1111 | F |

### Binary ↔ Hex Conversion

```
Binary:  1101 1011 0010 1111
Hex:     D    B    2    F

Result: 0xDB2F
```

## 3. Real-World Uses

| Use | Example | Why Hex |
|-----|---------|---------|
| **Colors** | `#FF0000` = red | 2 hex digits per RGB channel |
| **Memory** | `0x7FFF` = address | Compact for 16/32/64-bit values |
| **MAC addresses** | `00:1A:2B:3C:4D:5E` | 6 pairs of hex digits |

## 5. Summary

- **Hexadecimal** = base-16, digits 0-9 and A-F
- 4 binary digits = 1 hex digit
- Used for: **colors, memory addresses, MAC addresses**
