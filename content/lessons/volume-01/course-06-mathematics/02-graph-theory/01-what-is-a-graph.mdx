## 1. Hook

Open your social media app. Look at your friends list. Now look at mutual friends. Then suggested friends.

Every single one of those is a **graph problem**.

- Friends = vertices
- Friendships = edges
- Mutual friends = connected vertices
- Suggestions = "friends of friends"

Facebook processes this for 3 billion people. Maps do it for roads. Google does it for web pages. All of them use graphs.

## 2. Why This Matters

Without graphs, there would be no:
- Social networks (no "people you may know")
- GPS navigation (no shortest route)
- Internet search (no PageRank)
- Knowledge graphs (including LupaLearn's own!)
- Recommendation systems (no "users also bought")

Graphs are the hidden structure behind almost every connected system you use.

## 4. Core Concept

A **graph** is a collection of **vertices** connected by **edges**.

```
     ① —— ②
      |    |
     ③ —— ④
```

That's it. Two things:

| Part | What It Is | Real-World Example |
|------|------------|-------------------|
| **Vertex** (node) | A point or entity | A person, a city, a web page |
| **Edge** | A connection between two vertices | A friendship, a road, a hyperlink |

The power of graphs comes from how simple this is. With just vertices and edges, you can model almost any relationship.

### Directed vs Undirected

**Undirected** — edges go both ways (like a two-way road)

```
① —— ②   (① connects to ②, ② connects to ①)
```

Example: Facebook friends — if Alice is friends with Bob, Bob is friends with Alice.

**Directed** — edges have a direction (like a one-way street)

```
① → ②   (① connects to ②, but ② does NOT connect to ①)
```

Example: Twitter follows — Alice can follow Bob without Bob following Alice.

### Degree

The **degree** of a vertex is the number of edges connected to it.

```
     ① —— ②
      |    |
     ③ —— ④
```

Vertex ① has degree 2 (connected to ② and ③).
Vertex ④ has degree 2 (connected to ② and ③).

## 5. Mental Model

Think of a **city map**.

```
      🏠 City A —— 🏠 City B
           \          /
            🏠 City C
           /          \
      🏠 City D —— 🏠 City E
```

| Graph Concept | Map Concept |
|--------------|-------------|
| Vertex | A city |
| Edge | A road connecting two cities |
| Directed edge | A one-way street |
| Undirected edge | A two-way road |
| Degree | Number of roads from a city |
| Weighted edge | Distance between cities |

## 8. Programming Examples

Graphs don't have a built-in type in most languages, but they're easy to build.

### JavaScript — Adjacency List

```javascript
// A graph as an adjacency list
const graph = {
  Alice: ['Bob', 'Charlie'],
  Bob: ['Alice', 'David'],
  Charlie: ['Alice', 'David'],
  David: ['Bob', 'Charlie']
}

// Check if two people are connected
function areConnected(graph, person1, person2) {
  return graph[person1]?.includes(person2)
}

console.log(areConnected(graph, 'Alice', 'Bob'))    // true
console.log(areConnected(graph, 'Alice', 'David'))  // false (not directly)
```

### Python

```python
# A graph as a dictionary of sets
graph = {
    'Alice': {'Bob', 'Charlie'},
    'Bob': {'Alice', 'David'},
    'Charlie': {'Alice', 'David'},
    'David': {'Bob', 'Charlie'}
}

# Check connection
'Alice' in graph['Bob']  # True
'David' in graph['Alice']  # False
```

## 10. Common Mistakes

| Mistake | Why It's Wrong | Correct Understanding |
|---------|---------------|----------------------|
| "A graph is a bar chart" | Bar charts visualize data; graphs model relationships | Graphs are networks of connected things |
| "All graphs are trees" | Trees are a special type of graph with no cycles | A tree is a graph, but not all graphs are trees |
| "Edges must have weight" | Weights are optional | Most graphs start unweighted |
| "A vertex with no edges is useless" | Isolated vertices are still valid | A user with no friends is still a vertex |
| "Directed means the same as undirected" | Direction fundamentally changes the relationship | `A → B` is not the same as `A ← B` |

## 13. Summary

- A **graph** is a collection of **vertices** connected by **edges**
- **Undirected** edges go both ways (Facebook friends)
- **Directed** edges go one way (Twitter follows)
- **Degree** = number of edges connected to a vertex
- Graphs model relationships: social networks, maps, the web, knowledge graphs
- You can build a graph with just a dictionary of connections
