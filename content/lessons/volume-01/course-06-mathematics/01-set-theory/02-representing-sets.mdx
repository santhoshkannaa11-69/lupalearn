## 1. Hook

You know what a set is. Now how do you write one down so another person — or computer — understands exactly what you mean?

Two programmers on different continents need to agree on what `{1, 2, 3}` means. They need a standard way to write sets. That's what this lesson is about.

## 2. Why This Matters

Notation is how we communicate ideas precisely. In math, in code, in documentation — you need to write sets clearly. Every textbook, research paper, and API reference uses the same notation. Learn it once, read it everywhere.

## 3. Learning Objectives

- Write a set using roster notation (listing elements)
- Write a set using set-builder notation (describing elements)
- Recognize the empty set symbol (∅)
- Calculate the cardinality (size) of a set

## 4. Core Concept

There are two main ways to write a set.

### Roster Notation — List Everything

```
A = {1, 2, 3, 4, 5}
B = {apple, banana, orange}
```

Just list the elements inside curly braces `{}`, separated by commas.

<Callout type="tip">
  Think of roster notation like a grocery list — you write down every item you need.
</Callout>

### Set-Builder Notation — Describe the Pattern

```
A = {x | x is an even number between 1 and 10}
A = {2, 4, 6, 8, 10}
```

The left side says "all elements x such that" and the right side describes the condition.

```
{x | condition that x must satisfy}
```

Read the vertical bar `|` as "such that."

### Empty Set

A set with no elements. Two ways to write it:

```
∅
{ }
```

<Callout type="warning">
  `{∅}` is NOT the empty set — it's a set containing the empty set, which is a different thing!
</Callout>

### Cardinality — The Size of a Set

The number of elements in a set is called its **cardinality**.

```
|{1, 2, 3}|  = 3
|{a, b, c, d}|  = 4
|∅|  = 0
```

## 5. Mental Model

Think of roster notation as a **photo** of the set — you see every element.

Think of set-builder notation as a **description** — "show me everything that matches this description."

```
Roster:     {2, 4, 6, 8, 10}
Set-builder: {x | x is even and 1 < x < 11}
```

Both describe the same set. One shows you. One tells you.

## 7. Real-World Examples

| Situation | Notation | Why It Works |
|-----------|----------|-------------|
| All US states | Roster: {AL, AK, AZ, ..., WY} | Finite, known list |
| Even numbers | Set-builder: {x | x is even} | Infinite, needs description |
| Students present today | Roster: {Alice, Bob, ...} | Known at runtime |
| Prime numbers | Set-builder: {x | x has exactly 2 divisors} | Describes a rule |

## 8. Programming Examples

### JavaScript — No "roster" equivalent, but arrays are close

```javascript
// Roster-like
let fruits = new Set(['apple', 'banana', 'orange'])

// Cardinality
fruits.size  // 3

// Empty set
let empty = new Set()
empty.size  // 0
```

### Python

```python
# Roster-like
fruits = {'apple', 'banana', 'orange'}

# Cardinality
len(fruits)  # 3

# Empty set
empty = set()
len(empty)  # 0
```

### The Pattern in Code

```python
# Roster: list all elements
students_present = {'Alice', 'Bob', 'Charlie'}

# Set-builder: describe the condition
# (comprehension syntax)
even_numbers = {x for x in range(1, 11) if x % 2 == 0}
# {2, 4, 6, 8, 10}
```

## 10. Common Mistakes

| Mistake | Why It's Wrong | Correct Version |
|---------|---------------|-----------------|
| `{∅}` is the empty set | `{∅}` contains one element (the empty set) | The empty set is `∅` or `{}` |
| Using `[]` for sets | Square brackets mean arrays/lists | Sets use braces: `{1, 2, 3}` |
| `{1, 1, 2, 3}` is valid notation | Duplicates shouldn't be listed | Write `{1, 2, 3}` |
| "Cardinality is the number of unique vs total elements" | In sets, these are always the same | `|{1, 1, 2}|` = 3 if it were a list, but in a set the 1 is listed once so cardinality is 2 |
| Confusing ∅ with {∅} | `∅` has 0 elements; `{∅}` has 1 element | They are completely different sets |

## 13. Summary

- **Roster notation**: `{1, 2, 3}` — list every element
- **Set-builder notation**: `{x | condition}` — describe the rule
- **Empty set**: `∅` or `{}` — has zero elements (but `{∅}` is NOT empty!)
- **Cardinality**: `|A|` — the number of elements in set A
- Curly braces `{}` for sets, square brackets `[]` for lists/arrays
