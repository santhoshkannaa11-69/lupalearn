## 1. Hook

You've learned the math. Now here's the secret: every time a senior developer writes cleaner, faster code than a junior, there's a good chance they're using a Set instead of an Array.

A junior writes:

```javascript
let unique = []
for (let item of list) {
  if (!unique.includes(item)) unique.push(item)
}
```

A senior writes:

```javascript
let unique = [...new Set(list)]
```

One line. Faster. Clearer. That's the power of Sets in programming.

## 2. Why This Matters

Sets aren't just math — they're a practical tool that makes your code:

- **Faster**: `set.has()` is O(1), `array.includes()` is O(n)
- **Cleaner**: one line instead of loops with conditions
- **Safer**: no accidental duplicates, no index confusion
- **More expressive**: your code says exactly what it means

## 4. Core Concept

### Set vs Array — When to Use Which

| Need | Use | Why |
|------|-----|-----|
| Unique items | **Set** | Automatically rejects duplicates |
| Fast membership check | **Set** | O(1) vs O(n) for arrays |
| Order matters | **Array** | Sets have no order |
| Need index/position | **Array** | `array[2]` works, `set[2]` doesn't |
| Duplicates allowed | **Array** | Sets reject duplicates |
| Need both unique + ordered | **Array** + validate uniqueness | Or use a Set then convert to Array |

### The Golden Rule

> If you only need to know "is this in the collection?" — use a Set.
> If you need to know "where is it in the collection?" — use an Array.

### Performance Comparison

Searching for an item in a collection of 10,000 items:

| Data Structure | Operation | Time |
|---------------|-----------|------|
| Array | `array.includes(x)` | ~10,000 checks (slow) |
| Set | `set.has(x)` | ~1 check (instant) |

The difference grows as your data grows. At 100,000 items, an array search takes 100,000 steps. A Set still takes 1.

## 5. Mental Model

Think of a **VIP list for a club**.

The VIP list is a Set — the bouncer checks names instantly, no duplicates.

The general waiting list is an Array — order matters, duplicates possible (same name entered twice).

```
VIP List (Set):      {Alice, Bob, Charlie}
  → "Is Alice here?"      → Yes (instant)
  → "Is David here?"      → No (instant)

Waiting List (Array): [Alice, Bob, Alice, David, Eve]
  → "Is Alice here?"      → Yes (but had to scan through!)
  → "Where is Alice?"     → Position 0 (possible!)
```

## 8. Programming Examples

### JavaScript — Practical Patterns

```javascript
// 1. Deduplication
let withDupes = [1, 2, 2, 3, 3, 3, 4]
let unique = [...new Set(withDupes)]
// [1, 2, 3, 4]

// 2. Fast membership check
let blockedIps = new Set(['192.168.1.1', '10.0.0.5'])
function isBlocked(ip) {
  return blockedIps.has(ip)  // O(1)
}

// 3. Set operations
let A = new Set([1, 2, 3, 4])
let B = new Set([3, 4, 5, 6])

let union = new Set([...A, ...B])                    // {1,2,3,4,5,6}
let intersection = new Set([...A].filter(x => B.has(x)))  // {3,4}
let difference = new Set([...A].filter(x => !B.has(x)))   // {1,2}

// 4. Track unique visitors
let visitors = new Set()
function visit(userId) {
  visitors.add(userId)
  return visitors.size  // total unique visitors
}
```

### Python — Practical Patterns

```python
# 1. Deduplication
with_dupes = [1, 2, 2, 3, 3, 3, 4]
unique = list(set(with_dupes))
# [1, 2, 3, 4]

# 2. Fast membership
blocked_ips = {'192.168.1.1', '10.0.0.5'}
def is_blocked(ip):
    return ip in blocked_ips  # O(1)

# 3. Set operations (built-in!)
A = {1, 2, 3, 4}
B = {3, 4, 5, 6}

A | B  # union: {1,2,3,4,5,6}
A & B  # intersection: {3,4}
A - B  # difference: {1,2}
A ^ B  # symmetric difference: {1,2,5,6}

# 4. Unique word counter
def unique_words(text):
    return len(set(text.lower().split()))

print(unique_words("the cat and the dog and the bird"))
# 5 (the, cat, and, dog, bird)
```

## 10. Common Mistakes

| Mistake | Why It's Wrong | Correct Approach |
|---------|---------------|-----------------|
| Using an array for membership checks | O(n) instead of O(1) for large data | Use `Set.has()` |
| Using a Set when order matters | Sets don't preserve insertion order reliably | Use an Array (or convert Set → Array) |
| Forgetting to convert Set to Array | Many APIs expect arrays | `[...mySet]` in JS |
| Using a Set for everything | Sets can't do indexing, sorting, or duplicates | Choose the right tool for each job |
| Not knowing Set operations exist | Writing manual loops instead of one-liners | Use `filter` with `has` for intersection |

## 14. Summary

- Use **Sets** for: uniqueness, membership checks, deduplication, set operations
- Use **Arrays** for: ordered data, indexed access, duplicates allowed
- `set.has(x)` is **O(1)** — instant, even for millions of items
- `array.includes(x)` is **O(n)** — gets slower as data grows
- Common patterns: `[...new Set(arr)]` (dedup), `arr.filter(x => set.has(x))` (intersection)
- Set operations in code: union (spread + new Set), intersection (filter + has), difference (filter + !has)
