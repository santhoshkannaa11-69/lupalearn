## 1. Hook

Every time you see a point on a map, a cell in a spreadsheet, or a pixel on a screen — you're looking at a Cartesian product.

(X, Y) coordinates. (Row, Column) in a table. (Latitude, Longitude) on a map. All of them are **ordered pairs** from the Cartesian product of two sets.

## 2. Why This Matters

Cartesian products are the foundation of:

- **Coordinates and graphics**: every pixel is an (x, y) pair
- **Databases**: every row is a combination of column values
- **Functions**: input-output pairs
- **Matrices**: arranged in rows × columns
- **Relationships**: (student, course) pairs in enrollment data
- **Grids**: any 2D structure in programming

## 4. Core Concept

The **Cartesian product** of two sets A and B (written A × B) is the set of **all possible ordered pairs** where the first element comes from A and the second from B.

```
A × B = {(a, b) | a ∈ A, b ∈ B}
```

### Example

```
A = {1, 2}
B = {x, y}

A × B = {(1, x), (1, y), (2, x), (2, y)}
```

Each element of A pairs with each element of B. Like a multiplication table.

### Size

```
|A × B| = |A| × |B|
```

If A has 3 elements and B has 4:
|A × B| = 3 × 4 = 12 ordered pairs

### Order Matters

```
(1, x) is NOT the same as (x, 1)
```

The first element comes from A, the second from B. Swap them and you get a different pair.

```
A × B ≠ B × A  (unless A = B)
```

### More Than Two Sets

You can take the Cartesian product of any number of sets:

```
A × B × C = {(a, b, c) | a ∈ A, b ∈ B, c ∈ C}
```

## 8. Programming Examples

### JavaScript — Cartesian Product

```javascript
function cartesianProduct(A, B) {
  let result = []
  for (let a of A) {
    for (let b of B) {
      result.push([a, b])
    }
  }
  return result
}

let sizes = ['S', 'M', 'L']
let colors = ['Red', 'Blue']
console.log(cartesianProduct(sizes, colors))
// [['S','Red'], ['S','Blue'], ['M','Red'], ['M','Blue'], ['L','Red'], ['L','Blue']]
```

### Python — Using itertools

```python
from itertools import product

sizes = ['S', 'M', 'L']
colors = ['Red', 'Blue']

list(product(sizes, colors))
# [('S','Red'), ('S','Blue'), ('M','Red'), ('M','Blue'), ('L','Red'), ('L','Blue')]

# Three sets
list(product([1, 2], ['a', 'b'], ['x', 'y']))
# (1,'a','x'), (1,'a','y'), (1,'b','x') ...
```

### SQL — CROSS JOIN

```sql
SELECT * FROM colors CROSS JOIN sizes;
-- Returns every color paired with every size
```

## 10. Common Mistakes

| Mistake | Why It's Wrong | Correct Version |
|---------|---------------|-----------------|
| "A × B = B × A" | Ordered pairs make (1, x) different from (x, 1) | A × B ≠ B × A unless A = B |
| "Cartesian product is like union" | Union combines elements, product pairs them | |A × B| = |A| × |B|, not |A| + |B| |
| "(a, b) is the same as {a, b}" | (a, b) is an ordered pair; {a, b} is a set | Order matters in pairs, not in sets |
| "Empty set × A has elements" | If either set is empty, the product is empty | ∅ × A = ∅ |

## 13. Summary

- **A × B** = all ordered pairs (a, b) where a ∈ A and b ∈ B
- **|A × B|** = |A| × |B|
- **(a, b) ≠ (b, a)** — order matters in pairs
- **A × B ≠ B × A** in general
- If **either set is empty**, the product is empty
- Cartesian products are the foundation of coordinates, spreadsheets, databases, and combinations
