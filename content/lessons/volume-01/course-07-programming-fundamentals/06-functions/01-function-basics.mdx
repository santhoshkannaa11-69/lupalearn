## 1. Why This Matters

Functions are the building blocks of all code. They let you write a piece of logic once and use it everywhere.

## 2. Explanation

```javascript
function functionName(parameter1, parameter2) {
  // code to run
  return result;
}
```

| Part | Purpose |
|------|---------|
| `function` | Keyword that declares a function |
| `functionName` | How you refer to the function |
| `parameters` | Inputs the function receives |
| `body` | The code that runs |
| `return` | The output value |

## 3. Code Example

```javascript
// Define a function
function greet(name) {
  return "Hello, " + name + "!";
}

// Call the function
let message = greet("Alice");
console.log(message);  // Hello, Alice!

// Function with multiple parameters
function add(a, b) {
  return a + b;
}

console.log(add(5, 3));  // 8
console.log(add(10, 20)); // 30
```

## 5. Summary

- Functions group code into reusable blocks
- Parameters are inputs, return is output
- Call a function by using its name + parentheses
- Functions make code reusable and testable
