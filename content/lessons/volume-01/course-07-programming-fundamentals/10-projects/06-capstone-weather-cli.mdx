## 1. Capstone Project

Build a complete weather CLI tool that demonstrates everything you've learned. This is the final project that ties together the entire course.

## 2. Requirements

1. **Data Storage** — store weather data for multiple cities
2. **CRUD Operations** — add, list, update, delete city data
3. **Analysis** — calculate averages, find extremes, trends
4. **Reporting** — formatted output with statistics
5. **Error Handling** — validate all inputs, handle edge cases
6. **File Persistence** — save/load data (using localStorage concept)

## 3. Features to Implement

```javascript
// Data structure
// { city: string, temperature: number, humidity: number, condition: string, date: string }

// Functions to implement:
// addReading(city, temp, humidity, condition) — adds a weather reading
// getCityData(city) — returns all readings for a city
// getAverageTemp(city) — average temperature for a city
// getHottestDay(city) — returns the hottest reading
// getCitySummary(city) — formatted summary of all data
// getAllCities() — list all cities with data
// removeCity(city) — removes a city and its data
// compareCities(city1, city2) — compares two cities
```
