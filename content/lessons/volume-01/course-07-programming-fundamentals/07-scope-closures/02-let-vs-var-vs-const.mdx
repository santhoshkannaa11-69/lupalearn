## 1. Explanation

JavaScript has three ways to declare variables — each with different scoping rules:

| Keyword | Scope | Can Reassign? | Hoisted? |
|---------|-------|---------------|----------|
| `let` | Block `{}` | Yes | No |
| `const` | Block `{}` | No | No |
| `var` | Function | Yes | Yes (undefined) |

## 2. Code Example

```javascript
// let — block scoped
if (true) {
  let x = 10;
  const y = 20;
  var z = 30;
}
// console.log(x);  // ❌ — x is block scoped
// console.log(y);  // ❌ — y is block scoped
console.log(z);      // ✅ — 30 (var ignores blocks!)

// const — cannot reassign
const PI = 3.14159;
// PI = 3;          // ❌ — TypeError

// But objects declared with const CAN be modified
const user = { name: "Alice" };
user.name = "Bob";   // ✅ — object properties can change
// user = {};        // ❌ — can't reassign the variable
```

## 4. Summary

- Use `const` by default — only use `let` when you need to reassign
- Never use `var` in modern JavaScript
- `let` and `const` are block scoped, `var` is function scoped
- `const` prevents reassignment, but object properties can still change
