## 1. Hook

JavaScript has three ways to declare a variable. Two of them are modern. One is a relic from the early days that causes bugs. Can you guess which?

## 2. Comparison

| Keyword | Scope | Can Reassign? | Hoisted? |
|---------|-------|---------------|----------|
| `let` | Block `{}` | Yes | No |
| `const` | Block `{}` | No | No |
| `var` | Function | Yes | Yes (as undefined) |

## 3. Code Walkthrough

```javascript
if (true) {
  let x = 10;
  const y = 20;
  var z = 30;
}
// console.log(x);  // ❌ Block scoped
// console.log(y);  // ❌ Block scoped
console.log(z);      // ✅ 30 — var ignores blocks!

const PI = 3.14;
// PI = 3;           // ❌ Can't reassign const

// But objects declared with const CAN be modified:
const user = { name: "Alice" };
user.name = "Bob";    // ✅ Allowed
// user = {};         // ❌ Can't reassign
```

## 4. Common Mistake

```javascript
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Outputs: 3, 3, 3 (one shared `var`)

for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Outputs: 0, 1, 2 (each iteration gets its own `let`)
```

## 5. Summary

- Use **`const`** by default
- Use **`let`** when you need to reassign
- Never use **`var`** in modern JavaScript
