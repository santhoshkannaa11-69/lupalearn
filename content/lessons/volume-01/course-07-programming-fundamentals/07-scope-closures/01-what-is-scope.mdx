## 1. Hook

You create a variable inside a function, then try to use it outside — and it's gone. Where did it go? It didn't disappear. It was never there to begin with. That's scope.

## 2. Why This Matters

Scope determines where a variable is accessible. Without scope rules, every variable would be visible everywhere — causing chaos when different parts of your code use the same name.

## 3. Mental Model

Think of scope like **rooms in a house**:

- **Global scope** = the living room (everyone can access it)
- **Function scope** = a bedroom (only accessible from inside)
- **Block scope** = a closet (only accessible from inside that block)

## 4. Code Walkthrough

```javascript
let global = "I'm everywhere";

function example() {
  let local = "I'm only inside";
  console.log(global); // ✅ Works
  console.log(local);  // ✅ Works
}

console.log(global);    // ✅ Works
// console.log(local);  // ❌ ReferenceError
```

## 5. Execution Visualization

```
Memory:
  global → "I'm everywhere"

example() runs:
  Memory:
    global → "I'm everywhere"
    local  → "I'm only inside"

After example() ends:
  Memory:
    global → "I'm everywhere"
    local  → (cleaned up)
```

## 6. Common Mistakes

| Mistake | Why | Fix |
|---------|-----|-----|
| Using a variable before declaration | Hoisting confusion | Declare at the top of scope |
| Expecting `var` to be block-scoped | `var` ignores `{}` | Use `let` or `const` |

## 7. Summary

- **Global scope**: visible everywhere
- **Local scope**: visible only inside a function or block
- Inner functions can access outer variables, but not vice versa
