## 1. Explanation

**Scope** determines where a variable is accessible in your code. Think of it as "visibility" — some variables are visible everywhere, others only in specific blocks.

```javascript
// Global scope — visible everywhere
let globalVar = "I'm global";

function example() {
  // Local scope — only visible inside this function
  let localVar = "I'm local";
  console.log(globalVar);  // ✅ Works
  console.log(localVar);   // ✅ Works
}

console.log(globalVar);    // ✅ Works
// console.log(localVar);  // ❌ Error: not defined
```

## 2. Code Example

```javascript
let message = "Hello";  // Global scope

function greet() {
  let name = "Alice";   // Local to greet()
  console.log(message + " " + name);  // Works
}

greet();                 // Hello Alice
console.log(message);    // Hello
// console.log(name);    // ❌ ReferenceError
```

## 4. Summary

- Scope determines where variables are accessible
- Global scope = visible everywhere
- Local scope = visible only in a specific block/function
- Inner functions can access outer variables, but not vice versa
