## 1. Hook — Spark Curiosity

Imagine building a calculator app. You want to add two numbers: 5 and 3. The result is 8. Simple.

Now imagine that instead of using the numbers 5 and 3 directly in your code, those numbers come from a text box where the user types them. You don't know what they'll type before they type it.

How does the computer hold onto a value it doesn't know yet?

## 2. Why This Matters

Without variables, every program would be rigid. You'd have to hard-code every single value:

```
console.log("Hello, Alice!");
console.log("Hello, Bob!");
console.log("Hello, Charlie!");
```

What if you wanted to greet 1000 people? What if someone's name changes?

Variables let you write **one** piece of code that works for **any** value. They're the difference between a machine that can only do one thing and a computer that can do anything.

## 5. Core Concept — Clear Explanation

A **variable** is a named container that stores a value in the computer's memory.

Every variable has three parts:

| Part | What It Is | Example |
|------|-----------|---------|
| **Name** | How you refer to it | `age` |
| **Value** | What it holds | `25` |
| **Type** | What kind of data | `number` |

### The Four Common Data Types

| Type | What It Stores | Examples |
|------|---------------|----------|
| **string** | Text (in quotes) | `"Alice"`, `"hello"`, `"123"` |
| **number** | Integers and decimals | `42`, `3.14`, `-7` |
| **boolean** | True or false | `true`, `false` |
| **undefined** | No value yet | `let x;` |

> **Quick distinction:** A phone number like `"021"` should be a string, not a number — otherwise the leading zero gets dropped.

## 6. Mental Model — Three Analogies

**The Box Analogy**

```
┌───────────┐
│  age 25   │  ← label on the outside
│           │     value on the inside
└───────────┘
```

You can look inside the box (`console.log(age)`) or replace what's inside (`age = 26`).

**The Locker Analogy**

Like a gym locker. The locker number (name) never changes, but you can take things out and put new things in (value).

**The Contact List Analogy**

Your phone contacts list maps names to phone numbers. "Mom" points to a number. Mom gets a new phone — you update the number. The name "Mom" stays the same, but the value changes.

## 7. Historical Context

Early programmers had to track memory addresses directly (like `0x1A3F`). In assembly language, you wrote things like `MOV 0x1A3F, 25` — "store 25 at address 0x1A3F."

High-level languages like JavaScript introduced **named variables** so humans could write `age = 25` instead of tracking raw memory addresses. This made programming accessible to millions more people.

## 8. Real-World Applications

| Application | How It Uses Variables |
|-------------|----------------------|
| **Video game** | Stores player health, score, position, inventory |
| **E-commerce** | Stores cart total, user name, shipping address |
| **Weather app** | Stores temperature, humidity, forecast data |
| **Social media** | Stores post text, like count, user profile |

Every single app you use relies on variables.

## 9. Code Walkthrough — Explain Every Line

```javascript
let name = "Alice";
```

| Part | Meaning |
|------|---------|
| `let` | "Computer, create a variable" |
| `name` | "Call it 'name'" |
| `=` | "Store this value inside it" |
| `"Alice"` | "The value 'Alice'" |
| `;` | "End of instruction" |

```javascript
let age = 25;
console.log(age);
```

Line 1: Creates `age` and stores 25.
Line 2: "Computer, look up what's inside `age` and print it."

## 10. Execution Visualization

```
BEFORE ANY CODE RUNS:
  Memory: [empty]

let name = "Alice";
  Memory: [name → "Alice"]

let age = 25;
  Memory: [name → "Alice", age → 25]

age = 26;
  Memory: [name → "Alice", age → 26]
  (The value changed — the label stayed the same)

console.log(age);
  Output: 26
```

## 11. Common Mistakes

| Mistake | Why It's Wrong | Correct |
|---------|---------------|---------|
| `let 1stPlace = "Alice";` | Names can't start with a digit | `let firstPlace = "Alice";` |
| `let my-name = "Bob";` | Hyphens are subtraction | `let myName = "Bob";` |
| `console.log(score);` before `let score = 5;` | Using a variable before creating it | Declare first, then use |
| `let count = "five"; count + 1` | Can't do math on strings | Use `let count = 5;` |

## 16. Summary

- **Variables** are labeled containers in memory that hold values
- Use `let` to create a variable, `=` to assign a value
- Four common types: **string** (text), **number**, **boolean**, **undefined**
- Variables can change — that's why they're called *variable*s
- Pick the right type for the right data
