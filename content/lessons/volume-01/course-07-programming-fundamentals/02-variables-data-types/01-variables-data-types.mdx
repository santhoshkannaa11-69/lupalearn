## 1. Learning Objective

By the end of this lesson, you will be able to:

- **Define** what a variable is in programming terms
- **Declare** variables and **assign** values to them
- **Identify** the four most common data types
- **Choose** the correct data type for different kinds of information
- **Modify** a variable's value as your program runs

## 2. Why This Matters

Every program you will ever write needs to **store** and **manipulate** data. Whether it's a simple calculator, a video game, or a web application — variables are how you tell the computer "keep this value, I'll need it later."

Think of variables as **labeled boxes** in the computer's memory. Without them, you'd have to type every number, name, or result manually each time. Variables make your code **dynamic**, **flexible**, and **reusable**.

## 3. Explanation

### What Is a Variable?

A **variable** is a named container that holds a value in the computer's memory. Every variable has three properties:

| Property | Meaning | Example |
|----------|---------|---------|
| **Name** | How you refer to the variable | `age`, `username`, `total` |
| **Value** | The data stored inside | `25`, `"Alice"`, `true` |
| **Type** | What kind of data it holds | number, string, boolean |

### Common Data Types

Different kinds of data need different types. Here are the four you'll use most:

| Type | What It Stores | Examples |
|------|---------------|----------|
| **int** | Whole numbers | `42`, `0`, `-7`, `1000` |
| **float** | Decimal numbers | `3.14`, `-0.5`, `99.99` |
| **string** | Text (in quotes) | `"hello"`, `"Alice"`, `"123"` |
| **bool** | True or false | `true`, `false` |

> **Pro tip:** Use the right type for the right job. Storing a phone number as a *number* loses leading zeros (`021` becomes `21`). Store it as a *string* instead.

## 4. Visual Mental Model

Imagine the computer's memory as a row of empty boxes, each with a unique address:

```
┌──────┬──────┬──────┬──────┬──────┐
│      │      │  25  │      │      │
│  0x1 │  0x2 │  0x3 │  0x4 │  0x5 │
│      │      │  age │      │      │
└──────┴──────┴──────┴──────┴──────┘
```

When you write `let age = 25;`, the computer:
1. Finds an empty box at address `0x3`
2. Labels it with the name `age`
3. Stores the value `25` inside

Now whenever you use `age` in your code, the computer looks in that box.

## 5. Code Example

```javascript
// Declaring variables with different data types
let name = "Alice";       // string — text inside quotes
let age = 25;             // number (int) — whole number
let height = 1.68;        // number (float) — decimal
let isStudent = true;     // boolean — true or false

// Reading variables
console.log(name);        // Output: Alice
console.log(age);         // Output: 25

// Modifying variables
age = 26;                 // age is now 26
console.log(age);         // Output: 26
```

## 7. Common Mistakes

| Mistake | Why It's Wrong | Correct |
|---------|---------------|---------|
| `let 1stPlace = "Alice";` | Variable names can't start with a number | `let firstPlace = "Alice";` |
| `let my-name = "Bob";` | Hyphens are not allowed in names | `let myName = "Bob";` (use camelCase) |
| `console.log(age)` before declaring `let age` | Variables must exist before you use them | Declare with `let` first |
| `let count = "five";` then `count + 1` | Can't do math on strings | Use `let count = 5;` |

## 11. Summary

- **Variables** are named containers that store values in memory
- **Data types** define what kind of data a variable holds
- Use `let` to declare variables that can change
- Choose the right type: `int`, `float`, `string`, or `boolean`
- Variable names must start with a letter, `_`, or `$`
