## 1. Explanation

Many problems can be solved with either recursion or iteration (loops). Each has trade-offs.

| Aspect | Recursion | Iteration |
|--------|-----------|-----------|
| **Code** | More concise for tree problems | Usually simpler |
| **Memory** | Uses call stack (can overflow) | Uses less memory |
| **Speed** | Slower (function call overhead) | Faster |
| **Best for** | Trees, graphs, divide-and-conquer | Simple repetitions |

## 2. Code Example

```javascript
// Sum 1 to n — iterative
function sumIterative(n) {
  let total = 0;
  for (let i = 1; i <= n; i++) {
    total += i;
  }
  return total;
}

// Sum 1 to n — recursive
function sumRecursive(n) {
  if (n <= 1) return n;
  return n + sumRecursive(n - 1);
}

console.log(sumIterative(10));  // 55
console.log(sumRecursive(10));  // 55
```

## 4. Summary

- Use iteration for simple linear problems
- Use recursion for tree, graph, and divide-and-conquer
- Recursion has call stack limits (can overflow)
- Both approaches can solve the same problems
