## 1. Hook

Imagine holding up a mirror to another mirror. You see an infinite reflection — reflections within reflections. Recursion is the programming version of that: a function that calls itself.

## 2. Core Concept

Every recursive function needs two parts:

| Part | What It Does | Example |
|------|-------------|---------|
| **Base case** | Stops the recursion | `if (n <= 1) return 1` |
| **Recursive case** | Calls itself | `return n * factorial(n - 1)` |

## 3. Code Walkthrough

```javascript
function factorial(n) {
  if (n <= 1) return 1;       // Base case
  return n * factorial(n - 1); // Recursive case
}

console.log(factorial(5)); // 120
```

### Execution Visualization

```
factorial(5)
  → 5 * factorial(4)
    → 4 * factorial(3)
      → 3 * factorial(2)
        → 2 * factorial(1)
          → 1 (base case reached)
        → 2 * 1 = 2
      → 3 * 2 = 6
    → 4 * 6 = 24
  → 5 * 24 = 120
```

## 4. Recursion vs Loops

| Aspect | Recursion | Loop |
|--------|-----------|------|
| **Code** | More concise for tree problems | More explicit |
| **Memory** | Uses call stack (can overflow) | Less memory |
| **Best for** | Trees, divide-and-conquer | Simple repetitions |

## 6. Common Mistakes

| Mistake | Why | Fix |
|---------|-----|-----|
| No base case | Infinite recursion → stack overflow | Always define when to stop |
| Base case never reached | Wrong condition | Test with small values first |

## 7. Summary

- Recursion = a function that **calls itself**
- Every recursive function needs a **base case** and a **recursive case**
- Start with the base case when writing recursive functions
