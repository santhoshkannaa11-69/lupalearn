## 1. Why This Matters

Some problems are naturally recursive — the solution contains a smaller version of itself. Recursion lets you solve these problems elegantly.

## 2. Explanation

**Recursion** is when a function calls itself. Every recursive function needs two parts:

1. **Base case** — the condition that stops the recursion
2. **Recursive case** — the function calling itself with a smaller input

```javascript
function countdown(n) {
  // Base case
  if (n <= 0) {
    console.log("Done!");
    return;
  }
  // Recursive case
  console.log(n);
  countdown(n - 1);
}
```

## 3. Code Example

```javascript
// Factorial: n! = n × (n-1) × (n-2) × ... × 1
function factorial(n) {
  // Base case
  if (n <= 1) return 1;
  // Recursive case
  return n * factorial(n - 1);
}

console.log(factorial(5));  // 120 (5×4×3×2×1)

// Visual trace of factorial(5):
// factorial(5) → 5 * factorial(4)
//             → 5 * 4 * factorial(3)
//             → 5 * 4 * 3 * factorial(2)
//             → 5 * 4 * 3 * 2 * factorial(1)
//             → 5 * 4 * 3 * 2 * 1 = 120
```

## 5. Summary

- Recursion = a function calling itself
- Always need a base case (stops) and recursive case (continues)
- Start with the base case when writing recursive functions
- Recursion is elegant for tree-like problems
