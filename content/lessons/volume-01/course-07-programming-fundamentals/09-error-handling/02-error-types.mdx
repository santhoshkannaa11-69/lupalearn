## 1. Explanation

JavaScript has several built-in error types. Catching specific types lets you handle each appropriately.

| Error Type | What It Means | Example |
|-----------|---------------|---------|
| `ReferenceError` | Variable doesn't exist | `console.log(x)` where x is undefined |
| `TypeError` | Wrong type operation | `null.length` |
| `SyntaxError` | Invalid code syntax | `JSON.parse("{bad}")` |
| `RangeError` | Value out of range | `new Array(-1)` |
| `Error` | Generic error | `throw new Error("message")` |

## 2. Code Example

```javascript
try {
  // Try different errors by uncommenting:
  // console.log(undefinedVariable);    // ReferenceError
  // null.length;                       // TypeError
  JSON.parse("{bad}");               // SyntaxError
} catch (error) {
  if (error instanceof ReferenceError) {
    console.log("Reference Error:", error.message);
  } else if (error instanceof TypeError) {
    console.log("Type Error:", error.message);
  } else if (error instanceof SyntaxError) {
    console.log("Syntax Error:", error.message);
  } else {
    console.log("Unknown Error:", error.message);
  }
}
```

## 4. Summary

- Different errors have different types
- Use `instanceof` to check error types
- Handle each error type appropriately
- Always provide meaningful error messages
