## 1. Hook

Your program will crash. Not if — when. A network request fails. A file doesn't exist. A user enters invalid data. Error handling is the difference between a program that crashes and one that handles the problem gracefully.

## 2. Core Concept

```javascript
try {
  // Code that might fail
  let result = riskyOperation();
} catch (error) {
  // Runs if an error occurs
  console.log("Caught:", error.message);
} finally {
  // Always runs (optional)
}
```

## 3. Code Walkthrough

```javascript
function divide(a, b) {
  if (b === 0) {
    throw new Error("Division by zero!");
  }
  return a / b;
}

try {
  console.log(divide(10, 2));  // 5
  console.log(divide(10, 0));  // Error thrown — rest of try skipped
  console.log("This won't run");
} catch (error) {
  console.log("Error:", error.message);
}
```

## 4. Execution Visualization

```
divide(10, 2) → returns 5 → prints "5"
divide(10, 0) → throws Error
  → execution jumps to catch block
  → prints "Error: Division by zero!"
  → skip rest of try block
```

## 6. Common Mistakes

| Mistake | Why | Fix |
|---------|-----|-----|
| Empty catch block | Silently ignores errors | Always log or handle |
| Throwing non-Error values | Loses stack trace | Always `throw new Error()` |

## 7. Summary

- `try` wraps code that **might fail**
- `catch` handles errors when they occur
- `finally` always runs (for cleanup)
- Use `throw new Error()` to create your own errors
