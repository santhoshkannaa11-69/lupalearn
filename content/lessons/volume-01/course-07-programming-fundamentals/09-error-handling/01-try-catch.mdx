## 1. Why This Matters

Programs encounter errors: files that don't exist, network failures, invalid input. Without error handling, these crash your program. With try/catch, you handle them gracefully.

## 2. Explanation

```javascript
try {
  // Code that might fail
  let result = riskyOperation();
  console.log(result);
} catch (error) {
  // Runs if an error occurs
  console.log("Something went wrong:", error.message);
} finally {
  // Always runs (optional)
  console.log("Cleanup");
}
```

## 3. Code Example

```javascript
function divide(a, b) {
  if (b === 0) {
    throw new Error("Division by zero!");
  }
  return a / b;
}

try {
  console.log(divide(10, 2));  // 5
  console.log(divide(10, 0));  // Throws error
  console.log("This won't run");  // Skipped
} catch (error) {
  console.log("Error:", error.message);
} finally {
  console.log("Done");  // Always runs
}
```

## 5. Summary

- `try` wraps code that might fail
- `catch` handles errors if they occur
- `finally` always runs (cleanup)
- Use `throw new Error()` to create your own errors
