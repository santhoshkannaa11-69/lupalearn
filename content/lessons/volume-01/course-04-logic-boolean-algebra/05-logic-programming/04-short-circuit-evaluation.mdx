## 1. Hook

In `a && b`, if `a` is false, the result is always false — so the computer doesn't bother evaluating `b`. This is called **short-circuit evaluation**, and it's a key optimization in every programming language.

## 2. How Short-Circuiting Works

```javascript
// AND (&&): stops at first false
false && anything → false (anything not evaluated)
true && something → something (evaluated)

// OR (||): stops at first true
true || anything → true (anything not evaluated)
false || something → something (evaluated)
```

## 3. Practical Uses

```javascript
// Safe navigation
let userName = user && user.name;
// If user is null, stops before accessing user.name

// Default values
let name = inputName || "Guest";
// If inputName is empty/falsy, uses "Guest"

// Conditional execution
isValid && submitForm();
// Only calls submitForm() if isValid is true
```

## 5. Common Mistake

```javascript
// ❌ Wrong — changes the logic
let result = a || b && c;   // Not the same as (a || b) && c

// ✅ Use parentheses
let result = (a || b) && c;
```

## 6. Summary

- **AND** stops at the first `false`
- **OR** stops at the first `true`
- Used for **safe navigation**, **default values**, and **conditional execution**
