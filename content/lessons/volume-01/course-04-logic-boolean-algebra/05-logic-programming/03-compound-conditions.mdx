## 1. Hook

"I'll go outside if the weather is good AND I have free time, OR if there's an emergency." Real decisions combine multiple conditions with AND, OR, and NOT — exactly like Boolean expressions.

## 2. Operator Precedence

```
1. NOT (!)        — highest priority
2. AND (&&)
3. OR (||)        — lowest priority
```

```javascript
let a = true, b = false, c = true;

// !a && b || c
// Step 1: !a = false
// Step 2: false && b = false
// Step 3: false || c = true

// Use parentheses to be clear:
let result = (!a && b) || c;
```

## 3. Real Example — Product Filter

```javascript
function passesFilter(product) {
  let matchesSearch = product.name.includes("laptop");
  let inPriceRange = product.price >= 500 && product.price <= 1500;
  let inStock = product.quantity > 0;

  return matchesSearch && inPriceRange && inStock;
}
```

## 5. Summary

- Use **parentheses** to make operator precedence clear
- Break complex conditions into **named Boolean variables**
- Compound conditions are just combinations of AND, OR, NOT
