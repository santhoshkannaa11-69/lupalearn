# Representing Sets

## One Sentence
Sets can be written by listing their elements (roster form) or by describing a property (set-builder form).

## Why Should Someone Care?
You already understand what a set is. Now you need to read and write them — this is the notation used in every textbook, every whiteboard, and every technical interview.

## Teaching Goals
**Know:** Roster notation, set-builder notation, empty set symbol (∅), cardinality
**Do:** Write a set in both roster and set-builder forms
**Stop believing:** That math notation is unnecessary for programming

## Best Analogy
⭐⭐⭐⭐ **Grocery List**
Roster form = writing every item: {milk, eggs, bread}
Set-builder form = describing the items: {x | x is a dairy product in my fridge}

## Visual Ideas
Roster: `A = {1, 2, 3, 4, 5}`
Set-builder: `A = {x | x ∈ ℕ, 1 ≤ x ≤ 5}`

## Common Mistakes
- Confusing { } with [ ] (sets vs arrays/lists)
- Forgetting that { } ≠ ∅ (empty set has a symbol)
- Writing duplicates in roster form

## Quiz Ideas
1. What is the cardinality of {a, b, c}?
2. Which is valid set notation: {1, 2, 3} vs [1, 2, 3]?
3. What does ∅ mean?
4. Write {1, 2, 3, 4, 5} in set-builder notation

## Challenge Ideas
1. Convert a set from roster to set-builder
2. Identify valid vs invalid set notations
3. Find the cardinality of given sets
